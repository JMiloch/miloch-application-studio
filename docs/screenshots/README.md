# Screenshots

The README references four screenshots by filename. Drop your PNGs in this folder using **exactly these names**:

| File | What it should show |
|---|---|
| `welcome.png` | The Welcome page with the Drop Installer zone visible. Ideally with a Recent Projects row underneath so the page doesn't look empty. |
| `capture.png` | The Capture page after dropping an MSI. The MAS Catalog hit card + Detected Variables + Silent Install tab visible if possible. Firefox or a Microsoft Edge MSI works well because both have obvious silent switches. |
| `pre-ship.png` | The Pre-Ship page with an active profile selected. Descriptions & categories + Detection method + the SCCM/Intune tab switcher visible. |
| `rollout.png` | The Rollout page with the ship queue populated and at least one Completed row + one in-flight or Recent row. The right-side event timeline drawer open would be a plus. |

**Format guidance:**
- PNG, sensible resolution — a page-width crop at 1600 × 1000 px reads well on both retina and standard-DPI viewers.
- Dark or light mode is up to you — the site itself flips, so either works. Consistency between the four screenshots is more important than the choice.
- No personal / customer data visible. If a real profile / tenant name is on screen, blur or replace it.
- No system tray, no OS chrome — crop to just the MAS window.

Once your PNGs are in place, `README.md` at the repo root will render them without further changes.
